*boilerplate.txt*	Syntax highlighting for boilerplate templates

==============================================================================
INTRODUCTION					*boilerplate*

boilerplate is a compile-time Rust template engine. This plugin highlights
boilerplate templates: text outside of blocks is highlighted as a host
language and Rust code inside blocks is highlighted as Rust.

The four block kinds are:

	{% … %}		code block
	{{ … }}		interpolation block
	%% …		code line, to end of line
	$$ …		interpolation line, to end of line

==============================================================================
USAGE						*boilerplate-usage*

Boilerplate templates share the extension of their output and have no
distinguishing marker, so highlighting is enabled on demand rather than by
automatic file type detection.

							*:Boilerplate*
:Boilerplate		Highlight the current buffer as a boilerplate
			template. The buffer's current 'filetype' is used as
			the host language outside of blocks and Rust is used
			inside blocks. Run again to turn highlighting off.

:Boilerplate {host}	As above, but use {host} as the host language
			instead of the buffer's 'filetype'.

The host language is sourced from "syntax/{host}.vim" on 'runtimepath', so any
{host} for which a syntax file exists may be used.
<
In Neovim, Rust code inside blocks is shown in italic with an offsetting
background color.
